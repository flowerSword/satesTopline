# 售前头条 SalesTopline — 页面设计 UCD 文档

> 本文档基于当前代码实现（`static/index.html` + `static/js/app.js` 电脑端、`static/mobile/` 手机端）梳理页面结构与交互设计，供产品/设计/研发对齐当前 UI 现状使用。

## 一、整体设计定位

- 电脑端面向办公场景，信息密度高，突出资料管理、审核、统计等后台能力。
- 手机端面向移动查阅场景，采用底部 Tab + 卡片流布局，强调"搜索找资料 / 分类浏览 / 详情下载"三步路径。
- 两端共享同一套后端 API（`static/js/api.js`）与角色体系（`user` / `presales_admin` / `super_admin`），但界面与信息架构完全独立实现。
- 主色调统一为蓝色系（电脑端 `#2563EB`，手机端 `#1B4FBB`），辅以橙色（AI 智能检索/热门标识）与语义化状态色（绿=通过/在线、黄=待审、红=驳回/删除）。

## 二、电脑端（PC）页面设计

### 2.1 整体框架

- **顶部导航栏（Topbar）**：固定顶部，蓝色背景，高 56px。左侧为品牌 Logo + 一级导航（首页、资料库，管理员额外可见"资料管理"及"管理 ▾"下拉菜单）；右侧为用户头像圆标，点击展开用户菜单（我的收藏、浏览/下载历史、我的反馈、修改密码、退出登录）。
- **管理下拉菜单**：分类管理、标签管理、公告/推荐、反馈处理、数据看板、下载明细、用户管理（仅 `super_admin` 可见）。
- **路由**：Hash 路由（`#/path?param=val`），由 `route()` 统一分发到各视图渲染函数；`#/materials/:id` 识别为详情页。
- **内容区**：`#view-root` 容器，最大宽度 1400px 居中，标准页面内边距 24px。

### 2.2 页面清单

| 页面 | 路由 | 说明 |
|---|---|---|
| 首页 | `#/home` | 管理员可见 KPI 卡片（资料总数/本月新增/下载量/活跃用户/待审核数）；搜索框 + AI 智能检索入口 + 热搜词；7 天热门下载榜（全部/产品/方案 Tab）；管理员置顶推荐区；智能推荐区；最新资料动态；公告横幅 |
| 资料库 | `#/materials` | 左侧分类树筛选（可展开折叠）+ 文件格式筛选（PDF/DOCX/XLSX/PPTX/ZIP/MP4/PNG）+ 按维度分组的标签筛选；主区工具栏含关键词搜索、排序（最新/最多查看/最多下载）、上传按钮；卡片网格列表，分页 20 条/页 |
| 资料详情 | `#/materials/:id` | 顶部标题/元信息（分类路径、上传人、发布时间、状态徽标）+ 收藏/下载/编辑/发新版本按钮；正文含简介、文件预览（PDF 内嵌 iframe，图片直显，其他类型提示下载查看）、反馈提交表单；右侧统计卡片（浏览/下载/收藏数、文件大小）与版本历史列表 |
| 搜索结果 | `#/search` | 支持关键词 / AI 语义两种模式切换 Tab；语义模式展示相关度百分比；结果列表点击进入详情；无结果空状态 |
| 我的收藏 | `#/me/favorites` | 收藏资料的卡片网格，分页，空状态提示 |
| 浏览/下载历史 | `#/me/history` | 浏览历史 / 下载历史两个 Tab，列表展示操作时间，各自独立分页 |
| 我的反馈 | `#/me/feedbacks` | 表格：关联资料、类型（建议/错误/疑问）、内容、状态（待处理/已回复/已关闭）、提交时间、回复内容 |
| 资料管理（管理员） | `#/admin/materials` | 搜索 + 分类/状态筛选 + 快捷状态 Tab（全部/待审核/已发布/已驳回/已下架/已归档）；"重建 AI 索引"按钮；表格支持通过/驳回/下架/重新上架/置顶/归档/编辑等行内操作 |
| 分类管理（管理员） | `#/admin/categories` | 三级分类树，展开/折叠，新增子分类/编辑/删除；内置一级分类不可删除、锁定的二级分类不可删除 |
| 标签管理（管理员） | `#/admin/tags` | 按维度（产品/行业/格式/状态/场景/其他）分组展示标签及使用次数，支持增删改 |
| 公告与推荐（管理员） | `#/admin/announcements` | 公告表格（标题/级别徽标/发布状态/有效期/操作）+ 首页推荐位管理（选择资料 + 置顶标记） |
| 反馈处理（管理员） | `#/admin/feedbacks` | 按状态 Tab 筛选，表格含材料/提交人/类型/内容/状态，弹窗回复并流转状态（待处理→已回复→已关闭） |
| 数据看板（管理员） | `#/admin/stats` | 核心指标卡片、分类分布条形图、热门资料 Top10、活跃用户榜、30 天浏览/下载趋势双柱图、操作日志表 |
| 下载明细（管理员） | `#/admin/download_logs` | 三种视图：明细（关键词+日期范围筛选，支持导出 CSV）、按用户汇总、按资料汇总 |
| 用户管理（超级管理员） | `#/admin/users` | 用户表格（账号/姓名/角色/启用状态），支持新增、编辑、重置密码（默认 123456）、启用/禁用 |

### 2.3 弹窗（Modal）

- **上传资料**：文件选择（≤500MB，多格式）、标题、分类（叶子分类路径）、版本号、简介、标签多选；提交后进入"待审核"状态。
- **编辑资料**：标题、分类、简介、标签（不含换文件）。
- **发布新版本**：文件选择 + 版本号 + 更新说明。
- **修改密码**：旧密码/新密码/确认密码，前端 SHA-256 加密后提交，成功后强制退出重新登录。

### 2.4 角色权限矩阵

| 功能 | user | presales_admin | super_admin |
|---|:---:|:---:|:---:|
| 浏览/搜索/收藏/历史/反馈/上传 | ✓ | ✓ | ✓ |
| 管理导航菜单 | | ✓ | ✓ |
| 资料审核/上下架/归档 | | ✓ | ✓ |
| 分类/标签管理 | | ✓ | ✓ |
| 公告/推荐管理 | | ✓ | ✓ |
| 反馈处理/数据看板/下载明细 | | ✓ | ✓ |
| 用户管理 | | | ✓ |

### 2.5 交互与视觉规范

- **搜索双模式**：关键词（GET）与 AI 语义（POST，返回相关度分值），语义检索失败时提示回退关键词结果。
- **审核流转**：待审核 → 通过/驳回；已上线可下架/置顶/归档；已下架可重新上架。状态用色块+文案区分（⏳待审/✓通过/✗驳回）。
- **色彩**：主蓝 `#2563EB`（悬停 `#1D4ED8`，浅蓝背景 `#DBEAFE`）、强调橙 `#F97316`、成功绿 `#10B981`、警示黄 `#F59E0B`、危险红 `#EF4444`，中性灰阶 `--gray-50`~`--gray-900`。
- **组件**：按钮分 Primary/Secondary/Ghost；状态徽标、标签按维度着色；卡片白底轻阴影，圆角 8–12px；表格表头浅灰底，行 hover 高亮。
- **字体**：系统字体栈优先中文（苹方/微软雅黑），正文 14px，标题 22px，辅助文字 12–13px。
- **响应式**：内容最大宽度 1400px 居中；1024px 断点下双栏网格收为单栏。

## 三、手机端（Mobile）页面设计

### 3.1 整体框架

- **底部 Tab 导航**（固定，56px，含安全区适配）：首页 → 分类 → ➕上传（居中大图标，无文字）→ 收藏 → 我的。当前 Tab 高亮为主蓝色。
- **首页顶部 Topbar**：蓝色背景，左侧品牌名"售前头条"，右侧搜索图标 + 消息铃铛，仅首页可见。
- **二级页头（Sub-top）**：搜索/上传/详情/历史/我的收藏 等页面使用统一蓝色返回栏（← + 标题 + 可选右侧操作）。
- **分类浏览专用头（Cat header）**：独立蓝色头部（← + 一级分类名），与二级页头视觉一致但用于分类浏览场景。
- **路由**：Hash 路由（`#/home`、`#/cat`、`#/search`、`#/detail/:id`、`#/upload`、`#/favorites`、`#/history`、`#/profile` 等），路由函数统一控制 Topbar/Tab 导航的显隐。

### 3.2 页面清单

| 页面 | 路由 | 说明 |
|---|---|---|
| 首页 | `#/home` | 一级分类横向 Tab（点击直接跳转分类页并预选该分类，见 3.3）；搜索框 + AI 智能检索按钮（橙色）+ 热搜词；7 天热门榜（全部/产品/方案 Tab）；公告横幅；管理员置顶/智能推荐卡片区；最新资料动态列表 |
| 分类浏览 | `#/cat`、`#/cat?cid=<id>` | 详见 3.3，独立设计的分类浏览交互 |
| 搜索 | `#/search?q=&mode=` | 无查询词时展示热门搜索标签；有查询词时展示结果列表（语义模式显示"✨ AI 语义结果"徽标 + 相关度百分比） |
| 资料详情 | `#/detail/:id` | 顶部文件图标+标题+元信息+标签；简介、版本说明、浏览/下载统计；底部固定操作栏：下载到本地 / 收藏切换 / 分享 |
| 上传 | `#/upload` | 拖拽/点击上传区 + 表单（名称、版本号、所属分类、简介、版本说明），提交后进入审核并跳转"我的" |
| 我的收藏 | `#/favorites` | 收藏列表，支持单条删除，空状态提示 |
| 历史记录 | `#/history?type=<dl|view>` | 下载记录 / 浏览历史两个 Tab，按日期分组展示 |
| 我的 | `#/profile` | 个人信息卡（头像/姓名/部门/角色）+ 下载/上传/收藏统计 + 菜单（我的上传、下载记录、浏览历史、我的收藏、消息通知、意见反馈[开发中]、退出登录） |
| 登录 | 未路由，鉴权失败时渲染 | 全屏蓝色背景 + 账号密码表单，密码前端 SHA-256 加密后提交 |

### 3.3 分类浏览页设计（近期按 UCD 重构）

- 进入分类浏览后隐藏全局 Topbar、首页一级 Tab 及次级筛选栏，改为**独立蓝色头部**（← 返回首页 + 当前一级分类名），底部 Tab 导航保持可见。
- 头部下方为**全宽二/三级筛选 Chip 栏**：先展示二级分类 Chip（含"全部"），选中后若有下级再展示三级 Chip，贴紧头部，层次更清晰。
- 主体为左右分栏：左侧 72px 宽一级分类竖排列表（用于快速切换一级分类，选中态蓝色左边框），右侧为资料列表（图标+标题+版本/下载量/日期+下载按钮），无资料时展示空状态插画。
- 该设计对应提交 `f0e9bfa`（重构手机端分类浏览交互）与 `d6e695e`（修复筛选参数 `category_id` 传递问题）。

### 3.4 首页一级分类 Tab 跳转行为

- 首页顶部一级分类 Tab 不再提供"全部"选项且不做原地筛选，点击任意分类 Tab 直接跳转 `#/cat?cid=<id>`，分类浏览页读取 `cid` 参数并预选对应一级分类（提交 `90a6799`）。

### 3.5 交互与视觉规范

- **搜索入口**：首页搜索框（关键词）、"✨ AI 智能检索"按钮（语义，橙色 `#E87722`，空输入时 Toast 提示）、热搜标签、Topbar 搜索图标、7 天热门榜条目（直达详情）。
- **Toast 提示**：统一顶部居中半透明黑底浮层，2.2 秒自动消失，用于校验提示、成功/失败反馈。
- **空状态**：图标+文案组合，覆盖无数据/无分类/无收藏/无历史/无搜索结果/加载失败等场景。
- **色彩变量**：主蓝 `--blue:#1B4FBB`、浅蓝 `--blue-l:#EBF0FA`、深蓝 `--blue-d:#0D2E7A`、AI 橙 `--orange:#E87722`，灰阶 50–800。
- **视觉风格**：扁平化 + 轻阴影，Chip/按钮多为圆角胶囊形（12–17px 圆角），文件类型大量使用 Emoji 图标（📄 PDF、📊 PPT、🎬 视频等）。
- **适配**：`viewport` 禁用缩放，使用 `100dvh` 及 `env(safe-area-inset-*)` 适配全面屏/刘海屏，滚动区域隐藏原生滚动条并启用惯性滚动。
- **待实现功能**：消息通知、意见反馈、个人中心设置入口、历史记录"清空"按钮目前均为占位（Toast 提示"功能开发中"或无点击响应）。

## 四、双端设计差异小结

| 维度 | 电脑端 | 手机端 |
|---|---|---|
| 导航模式 | 顶部导航栏 + 下拉菜单 | 底部 5 Tab 导航 |
| 信息密度 | 高（表格、多栏筛选、后台管理） | 低（卡片流、分步筛选） |
| 分类筛选 | 左侧可展开树 | 独立分类浏览页（一级侧栏 + 二/三级 Chip） |
| 管理功能 | 完整后台（审核/统计/用户/日志） | 不提供，仅上传 + 个人中心 |
| 主色 | `#2563EB` | `#1B4FBB` |
| 详情页操作 | 收藏/下载/编辑/发新版本（并列按钮） | 底部固定操作栏（下载/收藏/分享） |
